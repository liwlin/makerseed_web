# Roadmap

## Phase 0 - Completed

- Repository structure reset around `website/`, `miniprogram/`, `cloud/`, `content/`, `assets/`, `design/`, and `docs/`.
- Website production files moved under `website/`.
- GitHub Pages workflow limited to the `website/` artifact.
- Old mini program pages, WOWKIDS-derived tokens, old data sources and old icons removed.
- Validation script added for project structure and design-alignment checks.

## Phase 1 - Completed as MVP

- Student/parent browsing, course detail, booking, works/growth, profile and membership shell.
- Teacher daily schedule, students, check-in and feedback shell.
- Admin dashboard, booking, student, course, teacher, order and membership shell.
- Shared mock service layer and models prepared for CloudBase replacement.
- Mini program development version `0.0.2` uploaded through WeChat Developer Tools CLI.

## Phase 2 - Next

- CloudBase persistence for booking, students, teachers, orders, memberships and check-in records.
- WeChat login role binding and permission enforcement.
- Admin operational dashboards backed by real data.
- Teacher class feedback and student growth history.

## Phase 3 - Later

- Payments, invoices and order reconciliation.
- Membership store, class-hour packages and coupon flows.
- Parent feedback history and richer growth portfolio.
- Production monitoring and audit logs.
